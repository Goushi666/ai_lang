# Web 端设计 · 环境分析页面

## 1. 路由与组件

- 路由示例：`/dashboard/analysis`（环境监测子路由）或 `/analysis`。
- 主组件：`views/EnvironmentAnalysis.vue`（或 `views/dashboard/AnalysisTab.vue` 作为 Tab 内容）。

## 2. 目录（增量）

```
src/
├── api/
│   └── analysis.js           # getEnvironmentSummary(params)
├── views/
│   └── EnvironmentAnalysis.vue
```

## 3. 页面结构

- **顶栏**：时间范围选择器（复用 `el-date-picker` datetimerange，与 Dashboard 行为一致）。
- **可选**：设备选择 `el-select`（多设备场景）。
- **主区**：
  - 三列或卡片：**温度 / 湿度 / 光照** 的 min、max、avg、count；
  - 可选：`ECharts` 第二图 — 分桶序列（`xAxis` 桶标签，`series` 三条线或分组柱）。
- **摘要区**：根据 `summary_code` 映射图标与 `summary_hints` 列表展示。
- **操作**：「重新分析」按钮 → 调用 GET 或 POST；`loading` 状态绑定按钮与骨架屏。

## 4. 数据流

1. `onMounted`：用默认时间窗调用 `analysisApi.getSummary`。
2. 用户改时间窗 → 可防抖 300ms 后自动请求，或仅手动点击「重新分析」（产品二选一，建议手动 + 可选自动）。
3. 无数据：`summary_code === 'insufficient_data'` 或 HTTP 404 → `ElEmpty`。

## 5. 与 Agent 联动（可选）

- 「用助手解读」：`router.push({ name: 'Agent', query: { context: encodeURIComponent(JSON.stringify({ window, summary_code })) } })`，Agent 页 `onMounted` 读 query 注入首条 user 消息模板。

## 6. API 封装示例 (`api/analysis.js`)

```javascript
import request from '@/utils/request'

export const analysisApi = {
  getSummary: (params) =>
    request.get('/api/analysis/environment/summary', { params }),
  runAnalysis: (params) =>
    request.post('/api/analysis/environment/run', null, { params }),
}
```

路径以后端 OpenAPI 为准。
