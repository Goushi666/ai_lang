# 后端设计 · 目录结构

```
backend/
├── app/
│   ├── api/                    # API 路由
│   │   ├── v1/
│   │   │   ├── sensors.py      # 传感器接口
│   │   │   ├── alarms.py       # 告警接口
│   │   │   ├── vehicles.py     # 车辆控制接口
│   │   │   ├── devices.py      # 设备管理接口
│   │   │   ├── analysis.py     # 环境分析接口
│   │   │   └── agent.py        # 智能 Agent 对话接口
│   │   └── deps.py             # 依赖注入（若存在）
│   ├── agent/                  # Agent：工具、提示词、LLM 客户端（可选包）
│   │   ├── tools.py
│   │   ├── prompts.py
│   │   └── llm_client.py
│   ├── core/                   # 核心配置
│   │   ├── config.py           # 配置管理
│   │   ├── security.py         # 安全相关
│   │   └── mqtt.py             # MQTT 客户端
│   ├── models/                 # 数据模型（ORM）
│   ├── schemas/                # Pydantic 模型
│   ├── services/               # 业务逻辑
│   │   ├── sensor_service.py
│   │   ├── alarm_service.py
│   │   ├── vehicle_service.py
│   │   ├── analysis_service.py # 环境分析
│   │   └── agent_service.py    # Agent 编排
│   ├── repositories/           # 数据访问
│   ├── websocket/              # WebSocket
│   │   └── manager.py
│   ├── middleware/             # 中间件
│   │   └── cors.py
│   └── main.py                 # 应用入口
├── alembic/                    # 数据库迁移（启用 ORM 持久化后）
├── tests/                      # 测试
├── requirements.txt
└── .env
```

实际文件名以仓库为准；新增模块时保持 **api → services → repositories** 依赖方向。
