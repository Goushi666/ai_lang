# 后端设计 · 系统架构与技术选型

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    API层 (FastAPI)                       │
├─────────────────────────────────────────────────────────┤
│  路由层  │  中间件  │  WebSocket  │  认证授权             │
├─────────────────────────────────────────────────────────┤
│              业务逻辑层 (Service)                         │
├─────────────────────────────────────────────────────────┤
│  设备管理  │  数据处理  │  告警判定  │  车辆控制           │
├─────────────────────────────────────────────────────────┤
│              数据访问层 (Repository)                      │
├─────────────────────────────────────────────────────────┤
│  MySQL  │  Redis  │  MQTT Broker                        │
└─────────────────────────────────────────────────────────┘
```

## 2. 技术选型

- **Web 框架**：FastAPI 0.104+
- **数据库**：MySQL 8.0（目标形态；MVP 可用内存实现过渡）
- **缓存**：Redis 7.0（目标形态）
- **消息队列 / 物联网**：MQTT（如 Mosquitto 或云 IoT）
- **ORM**：SQLAlchemy 2.0
- **异步**：asyncio，及与 I/O 配套的异步客户端
- **认证**：JWT（python-jose）
- **数据验证**：Pydantic
- **部署**：Uvicorn（开发），生产可配合 Gunicorn 等

## 3. 与代码仓库对齐说明

实现阶段可在不违背对外契约的前提下，将「MySQL/Redis」替换为内存 Repository；本设计描述 **目标分层与扩展点**。
