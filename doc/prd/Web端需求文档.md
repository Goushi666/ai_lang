# Web端需求文档

> **文档索引**：本文件保留为 **Web 端总体需求**（项目级）。按功能模块拆分的详细需求见 [`frontend/`](./frontend/README.md)。

## 1. 项目概述

### 1.1 项目背景
本项目旨在开发井场环境监测与智能巡检系统的Web端应用，为用户提供实时环境数据监控、智能巡检车远程控制、视频监控及告警管理等功能。

### 1.2 技术栈
- 前端框架：Vue.js
- 通信协议：WebSocket（实时数据）、RESTful API（历史数据查询）
- 后端对接：FastAPI服务

---

## 2. 功能需求

### 2.1 环境监测数据看板

#### 2.1.1 实时数据展示
- 显示温度、湿度、光照强度等环境参数的实时数值
- 数据通过WebSocket实时推送，无需手动刷新
- 支持数据单位显示（℃、%RH、Lux等）

#### 2.1.2 历史数据查询
- 支持按时间范围查询历史环境数据
- 提供数据趋势图表展示（折线图）
- 支持数据导出功能（CSV格式）

#### 2.1.3 数据可视化
- 使用图表组件展示数据变化趋势
- 支持多参数对比显示
- 提供时间轴缩放与拖拽功能

---

### 2.2 视频巡检实时画面

#### 2.2.1 实时视频流
- 显示巡检车摄像头采集的实时视频画面
- 支持视频流的流畅播放（延迟<500ms）
- 视频分辨率自适应网络带宽

#### 2.2.2 视频控制
- 支持视频画面的暂停/播放
- 支持全屏显示模式
- 显示视频连接状态指示

---

### 2.3 告警信息中心

#### 2.3.1 实时告警推送
- 通过WebSocket接收后端推送的告警事件
- 告警信息包含：告警类型、告警级别、触发时间、具体参数值
- 支持多级告警（低、中、高、紧急）

#### 2.3.2 告警展示
- 告警列表按时间倒序排列
- 不同告警级别使用不同颜色标识
- 支持告警消息的已读/未读状态管理

#### 2.3.3 告警历史
- 支持查询历史告警记录
- 支持按告警级别、时间范围筛选
- 支持告警统计分析（按类型、级别统计）

#### 2.3.4 告警通知
- 新告警触发时页面弹窗提示
- 支持浏览器桌面通知（需用户授权）
- 支持告警声音提示（可配置开关）

---

### 2.4 智能车远程控制面板

#### 2.4.1 手动控制模式
- 提供方向控制按钮：前进、后退、左转、右转、停止
- 支持键盘快捷键控制（WASD或方向键）
- 显示当前控制模式状态（手动/自动）

#### 2.4.2 自动循迹模式
- 提供自动循迹模式启动/停止按钮
- 显示循迹状态指示（运行中/已停止）
- 显示循迹偏差量与修正量（可选）

#### 2.4.3 运行状态监控
- 实时显示车辆速度信息
- 显示当前运行模式（手动/自动）
- 显示指令执行反馈状态
- 显示连接状态（在线/离线）

#### 2.4.4 安全保护
- 断连状态下禁用控制按钮
- 显示异常状态告警信息
- 提供紧急停止按钮（优先级最高）

---

### 2.5 系统配置管理

#### 2.5.1 设备管理
- 查看已接入设备列表
- 显示设备在线状态
- 支持设备基本信息配置

#### 2.5.2 告警阈值配置
- 配置各环境参数的告警阈值
- 支持多级阈值设置
- 配置修改后实时生效

#### 2.5.3 用户设置
- 用户登录/登出功能
- 个性化界面设置（主题、语言等）
- 通知偏好设置

---

### 2.6 环境分析

- 在选定时间窗内展示温度、湿度、光照的 **统计聚合**（如均值、最大、最小、采样点数）
- 支持可选 **分桶趋势**（如按小时）与 **结构化分析摘要**（结论代码 + 提示文案）
- 与环境监测数据 **同源**，不展示虚构数据；无数据时空态提示
- 详细需求见 [`frontend/09-环境分析.md`](frontend/09-环境分析.md)

### 2.7 智能 Agent 助手

- 提供 **对话式界面**，通过自然语言查询环境数据、告警、车辆状态及分析摘要（由后端工具访问真实接口）
- 支持多轮对话；可选 **流式输出**
- 车辆控制与敏感配置默认 **不通过助手直接执行**（若后续开放须二次确认）
- 详细需求见 [`frontend/10-智能Agent助手.md`](frontend/10-智能Agent助手.md)

## 3. 非功能需求

### 3.1 性能要求
- 页面首次加载时间 < 3秒
- WebSocket消息延迟 < 200ms
- 视频流延迟 < 500ms
- 支持同时在线用户数 ≥ 10

### 3.2 兼容性要求
- 支持Chrome、Firefox、Edge等主流浏览器（最新两个版本）
- 支持桌面端与平板设备
- 响应式布局适配不同屏幕尺寸

### 3.3 安全性要求
- 用户身份认证与授权
- 敏感操作需二次确认
- 通信数据加密传输（HTTPS/WSS）
- 防止XSS、CSRF等常见Web攻击

### 3.4 可用性要求
- 界面简洁直观，操作流程清晰
- 关键操作提供操作提示
- 异常情况提供友好的错误提示
- 支持操作撤销与确认机制

### 3.5 可维护性要求
- 代码结构清晰，模块化设计
- 关键功能提供注释说明
- 遵循Vue.js最佳实践
- 提供开发文档与部署文档

## 4. 接口需求

### 4.1 RESTful API接口

#### 4.1.1 环境数据接口
- `GET /api/sensors/history` - 查询历史环境数据
- `GET /api/sensors/latest` - 获取最新环境数据
- `GET /api/sensors/export` - 导出历史数据

#### 4.1.2 设备管理接口
- `GET /api/devices` - 获取设备列表
- `GET /api/devices/{id}` - 获取设备详情
- `PUT /api/devices/{id}` - 更新设备配置

#### 4.1.3 告警管理接口
- `GET /api/alarms/history` - 查询历史告警
- `PUT /api/alarms/{id}/read` - 标记告警已读
- `GET /api/alarms/config` - 获取告警配置
- `PUT /api/alarms/config` - 更新告警配置

#### 4.1.4 车辆控制接口
- `POST /api/vehicle/control` - 发送控制指令
- `GET /api/vehicle/status` - 获取车辆状态

#### 4.1.5 环境分析接口
- `GET /api/analysis/environment/summary` - 按时间窗（及可选设备、分桶）返回聚合与结构化摘要
- `POST /api/analysis/environment/run`（可选）- 触发一次分析计算并返回

#### 4.1.6 智能 Agent 接口
- `POST /api/agent/chat` - 非流式对话
- 流式端点（可选）：见模块化需求文档 [`backend/09-智能Agent.md`](backend/09-智能Agent.md)
- `GET /api/agent/health`（可选）- 助手服务是否可用

### 4.2 WebSocket接口

#### 4.2.1 实时数据推送
- 环境传感器数据实时推送
- 车辆运行状态实时推送
- 告警事件实时推送
- 环境分析摘要更新推送（可选，消息类型如 `analysis_summary`）

#### 4.2.2 连接管理
- 心跳保活机制
- 断线自动重连
- 连接状态通知

## 5. 界面原型

### 5.1 页面结构
- 顶部导航栏：系统标题、用户信息、退出登录
- 左侧菜单栏：环境监测、视频巡检、告警中心、车辆控制、系统设置
- 主内容区：对应功能页面内容
- 底部状态栏：系统状态、连接状态、版本信息

### 5.2 核心页面
1. 环境监测页面：数据卡片 + 趋势图表
2. 视频巡检页面：视频播放器 + 控制按钮
3. 告警中心页面：告警列表 + 筛选器 + 统计图表
4. 车辆控制页面：控制面板 + 状态监控 + 视频预览
