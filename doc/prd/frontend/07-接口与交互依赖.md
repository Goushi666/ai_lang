# Web 端需求 · 接口与交互依赖

## 1. RESTful API（摘要）

详细路径与参数与后端 OpenAPI 对齐；以下为需求级清单。

### 1.1 环境数据

- `GET /api/sensors/history` — 查询历史环境数据  
- `GET /api/sensors/latest` — 获取最新环境数据（无数据时后端可返回 404）  
- `GET /api/sensors/export` — 导出历史数据  

### 1.2 设备管理

- `GET /api/devices` — 获取设备列表  
- `GET /api/devices/{id}` — 获取设备详情  
- `PUT /api/devices/{id}` — 更新设备配置  

### 1.3 告警管理

- `GET /api/alarms/history` — 查询历史告警  
- `PUT /api/alarms/{id}/read` — 标记告警已读  
- `GET /api/alarms/config` — 获取告警配置  
- `PUT /api/alarms/config` — 更新告警配置  

### 1.4 车辆控制

- `POST /api/vehicle/control` — 发送控制指令  
- `GET /api/vehicle/status` — 获取车辆状态  

### 1.5 环境分析

- `GET /api/analysis/environment/summary` — 聚合与摘要（参数：`start_time`、`end_time`、可选 `device_id`、`bucket`）  
- `POST /api/analysis/environment/run` — 可选，显式触发分析  

### 1.6 智能 Agent

- `POST /api/agent/chat` — 多轮对话  
- 流式 / 健康检查 — 见 [`../backend/09-智能Agent.md`](../backend/09-智能Agent.md)  

## 2. WebSocket

### 2.1 实时数据推送

- 环境传感器数据实时推送（`sensor_data`）。  
- 车辆运行状态实时推送（`vehicle_status`）。  
- 告警事件实时推送（`alarm`）。  
- 可选：环境分析摘要（`analysis_summary`）。  

### 2.2 连接管理

- 心跳保活机制。  
- 断线自动重连。  
- 连接状态通知（界面层展示）。  

## 3. 界面结构（节选）

- 顶部导航栏：系统标题、用户信息、退出登录。  
- 左侧菜单栏：环境监测、环境分析（或并入环境监测）、视频巡检、告警中心、车辆控制、系统设置、智能助手（或全局入口）。  
- 主内容区：对应功能页面内容。  
- 底部状态栏：系统状态、连接状态、版本信息。  

## 4. 核心页面

1. 环境监测页面：数据卡片 + 趋势图表。  
2. 视频巡检页面：视频播放器 + 控制按钮。  
3. 告警中心页面：告警列表 + 筛选器 + 统计图表。  
4. 车辆控制页面：控制面板 + 状态监控 + 视频预览。  
5. 环境分析页面：时间窗 + 聚合指标 + 可选分桶图 + 摘要。  
6. 智能助手：对话区 + 输入与发送（抽屉或独立页）。
