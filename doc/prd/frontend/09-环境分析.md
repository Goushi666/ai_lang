# Web 端需求 · 环境分析

## 1. 目标

在 **环境监测** 能力之上，为用户提供 **选定时间范围内的环境数据统计、趋势摘要与简单结论展示**，与现有实时卡片、历史折线图 **互补**（分析侧重聚合与解读入口，不替代原始曲线）。

## 2. 功能需求

### 2.1 入口与导航

- 在 **环境监测** 相关区域提供入口，任选一种或组合：
  - 环境监测页内 **「环境分析」** 页签/折叠面板；
  - 或左侧菜单独立 **「环境分析」** 菜单项。
- 进入后默认时间窗与当前看板或日期选择器 **可联动**（如默认最近 24 小时）。

### 2.2 分析结果展示

- 展示后端返回的 **聚合指标**：各参数 min/max/avg/count（或等价字段）。
- **可选分桶图表**（如按小时柱状/折线），与后端 `bucket` 参数对齐。
- **结构化摘要**：用卡片或列表展示结论代码对应的中文说明（如「温度整体平稳」「湿度持续偏高」）；若无数据，展示 **空态**，不展示虚构数值。

### 2.3 交互

- **刷新分析**：用户变更时间范围或设备后，可点击「重新分析」触发请求（对应后端 GET 或 POST）。
- 加载中与失败态：**Loading**、**错误提示**（含超时、无权限、服务关闭等）。

### 2.4 与 Agent 的衔接（可选）

- 提供 **「用助手解读」** 快捷按钮：将当前时间窗或摘要 ID 传给 Agent 对话（打开抽屉/跳转 Agent 页并预填上下文），具体参数与 [`10-智能Agent助手.md`](./10-智能Agent助手.md) 约定。

## 3. 接口与实时通道

- **REST**：`GET /api/analysis/...`（以后端 OpenAPI 为准，见 [`../backend/08-环境分析.md`](../backend/08-环境分析.md)）。
- **WebSocket**：若后端推送 `analysis_summary`，本页可订阅并提示「分析已更新」；首版可仅轮询或手动刷新。

## 4. 非功能

- 分析请求期间避免阻塞整页：异步请求 + 骨架屏或局部 Loading。
- 大屏或平板上布局可读，关键数字字号与对比度符合现有设计规范。
