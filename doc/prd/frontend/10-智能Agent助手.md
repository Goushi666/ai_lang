# Web 端需求 · 智能 Agent 助手

## 1. 目标

提供 **对话式智能助手**，帮助用户用自然语言 **查询环境数据、告警与车辆状态、理解环境分析结果**；默认 **不直接执行** 车辆控制与敏感配置（除非产品后续开放并加二次确认）。

## 2. 功能需求

### 2.1 入口与布局

- 全局 **悬浮按钮** 或 **顶栏图标** 打开助手面板（抽屉 / 侧栏 / 独立页面三选一，需移动端可用）。
- 独立路由 **「智能助手」** 可选，用于大屏深度对话。

### 2.2 对话能力

- **多轮对话**：展示用户与助手气泡列表，支持滚动与自动滚到底部。
- **流式输出**（若后端支持 SSE）：逐字或逐段显示；若仅非流式，则整段显示并配合 Loading。
- **清空会话**、**复制最后一条回复**（可选）。

### 2.3 上下文增强（可选）

- 从 **环境监测 / 分析页** 跳转时，可携带 **当前时间范围、设备 ID、摘要片段** 作为首条 system/user 隐藏上下文，减少用户重复描述。

### 2.4 错误与降级

- 后端 `AGENT_ENABLED=false` 或网络错误时：**禁用入口或展示「暂不可用」**，提示简洁可懂。
- **超时**：提示重试，不无限转圈。

### 2.5 安全提示（界面）

- 在助手面板显著位置提示：**助手通过系统工具读取数据，重要操作请以控制台为准**；若未来开放写工具，须 **二次确认弹窗**。

## 3. 接口依赖

- `POST /api/agent/chat`（必填，首版）
- 可选：`GET/POST` 流式端点（见 [`../backend/09-智能Agent.md`](../backend/09-智能Agent.md)）

## 4. 与 WebSocket

- 首版 **不强制** 使用现有 `/ws`；若统一连接，需遵循后端约定的 `type`（如 `agent_delta`），与传感器推送分流处理。

## 5. 非功能

- 助手面板打开时不应阻塞其他页面操作（模态策略慎用）。
- 无障碍：输入框可聚焦、按钮可键盘操作。
