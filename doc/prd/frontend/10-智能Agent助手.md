# Web 端需求 · 智能 Agent 助手

## 0. 文档说明

- **关联文档**：后端 Agent 需求 [`../backend/09-智能Agent.md`](../backend/09-智能Agent.md)

---

## 1. 目标

提供 **对话式智能助手**，帮助用户用自然语言查询环境数据、理解分析结果、进行知识问答。支持渐进式追问交互与 RAG 模式切换。

## 2. 功能需求

### 2.1 入口与布局

- 全局 **悬浮按钮** 或 **顶栏图标** 打开助手面板（抽屉/侧栏），不阻塞其他页面操作
- 可选独立路由「智能助手」页面，用于大屏深度对话
- 移动端可用

### 2.2 对话能力

- **多轮对话**：展示用户与助手气泡列表，自动滚到底部
- **流式输出**：支持 SSE 逐段显示回答内容，配合打字机效果；降级为非流式时整段显示 + Loading
- **清空会话**、**复制回复**

### 2.3 渐进式追问交互（核心）

当后端返回 `clarification` 字段非空时，前端渲染追问 UI：

- 展示 Agent 的追问文本（`clarification.question`）
- 渲染 **选项按钮组**（`clarification.options`），用户点击即自动发送对应回复
- 若 `allow_custom=true`，同时保留文本输入框供用户自由回答
- 选项按钮样式区分于普通消息，建议使用 chip/tag 形态

**交互示例**：

```
用户：温度怎么样？

助手：您提到想了解温度情况，我想确认一下：
      您想查看哪个时间范围的温度数据？

      [最近 1 小时]  [今天]  [最近 24 小时]  [最近一周]

      或者您也可以直接输入具体的时间范围 ▁
```

### 2.4 对话模式切换

- 在助手面板顶部或设置区提供 **模式切换**（通用对话 / 知识问答）
- 切换时提示：模式变更后当前上下文可能受影响
- RAG 模式下，回答中附带的引用来源渲染为可点击链接或展开卡片

### 2.5 上下文携带

- 从环境监测/分析页跳转时，可携带当前时间范围、设备 ID、摘要片段作为预填上下文
- 打开助手面板时自动注入，减少用户重复描述

### 2.6 错误与降级

- 后端 `AGENT_ENABLED=false` 或网络错误：禁用入口或展示"暂不可用"
- 超时：提示重试，不无限转圈
- 流式连接断开：展示已收到的内容 + 错误提示

### 2.7 安全提示

- 在助手面板显著位置提示：「助手通过系统工具读取数据，重要操作请以控制台为准」
- 若未来开放写工具，须二次确认弹窗

## 3. 接口依赖

- `POST /api/agent/chat`（必填，P0）：支持 `stream=true` 时返回 SSE
- `GET /api/agent/sessions/{id}`（P0）：恢复会话
- `DELETE /api/agent/sessions/{id}`（P0）：清除会话

## 4. 非功能

- 助手面板打开时不阻塞其他页面操作
- 无障碍：输入框可聚焦、按钮可键盘操作
- 对话列表虚拟滚动（长会话场景性能保障）
