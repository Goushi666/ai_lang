# Web 端需求 · 告警中心

## 1. 实时告警推送

- 通过 WebSocket 接收后端推送的告警事件。
- 告警信息包含：告警类型、告警级别、触发时间、具体参数值。
- 支持多级告警（低、中、高、紧急）。

## 2. 告警展示

- 告警列表按时间倒序排列。
- 不同告警级别使用不同颜色标识。
- 支持告警消息的已读/未读状态管理。

## 3. 告警历史

- 支持查询历史告警记录。
- 支持按告警级别、时间范围筛选。
- 支持告警统计分析（按类型、级别统计）。

## 4. 告警通知

- 新告警触发时页面弹窗提示。
- 支持浏览器桌面通知（需用户授权）。
- 支持告警声音提示（可配置开关）。

## 5. 与后端依赖

- REST：`GET /api/alarms/history`、`PUT /api/alarms/{id}/read`、`GET/PUT /api/alarms/config`。
- WebSocket：订阅告警类型消息（与后端约定字段一致）。
