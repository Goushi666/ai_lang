# 后端需求 · 非功能与演进

## 1. 非功能需求（与 Web 端协同）

- **延迟**：WebSocket 推送路径在局域网/典型部署下应满足 Web 端性能要求（参见 [`../Web端需求文档.md`](../Web端需求文档.md) 3.1）。
- **并发**：支持至少与 Web 端约定相当的并发在线连接数（如 ≥10），具体压测指标可在测试阶段量化。
- **可用性**：关键路径（MQTT 消费、WS 广播）异常时不应静默崩溃；应可记录日志并具备重启恢复策略。

## 2. 演进方向（设计文档对齐）

总体设计中的技术选型包含 **MySQL、Redis** 等；当前 MVP 可用内存实现替代，但需求上应满足：

- 传感器、告警等数据 **可迁移** 至关系型存储而不改变对外 API 语义；
- 缓存与会话（若引入）不破坏现有 JWT/REST 契约。

## 3. 可观测性

- 建议提供健康检查或最小就绪探针（可与 `GET /` 或独立路由结合），便于部署编排。

## 4. 交付与文档

- 与 Web 联调所需的 **接口列表、WebSocket 消息示例** 应在总体或接口专章中保持更新；
- 本模块需求不重复硬件交付物，硬件仍以 [`../硬件端需求文档.md`](../硬件端需求文档.md) 为准。
