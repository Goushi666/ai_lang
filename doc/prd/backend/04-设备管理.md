# 后端需求 · 设备管理

## 1. 目标

支持 Web 端「设备列表、在线状态、基本信息配置」等能力所需的后端接口（见 [`../Web端需求文档.md`](../Web端需求文档.md) 2.5.1）。

## 2. 接口需求（目标形态）

与 Web 端接口章节对齐时，应提供：

- `GET /api/devices`：分页或列表返回已接入设备概要（ID、名称、类型、在线状态等）；
- `GET /api/devices/{id}`：单设备详情；
- `PUT /api/devices/{id}`：更新可写配置项（名称、标签等，具体字段可迭代）。

## 3. 当前实现阶段说明

代码库在演进过程中可能仅提供 **占位或最小接口**（例如连通性探测）。本需求文档描述 **目标契约**；实现未齐备时，应在发布说明或总体设计中标注差距，避免前端按完整契约联调时产生歧义。

## 4. 验收要点（达标时）

- 列表与详情字段满足 Web 端展示与编辑表单；
- 更新接口具备基本校验与错误返回；
- 在线状态与 MQTT/心跳等数据源策略在总体设计中有说明。
