# 后端需求 · 安全与配置

## 1. 认证与授权

- 支持 **JWT** 配置（密钥、算法），为后续登录与受保护接口预留能力；MVP 阶段若接口开放，应在总体文档中声明风险与收紧计划。
- 敏感配置仅通过环境变量或密钥管理注入，不得写入仓库明文。

## 2. 跨域（CORS）

- 可配置允许的前端来源；开发环境可与 Vite 代理配合。
- WebSocket 与 HTTP 的跨域策略需与部署方式一致，避免浏览器握手失败。

## 3. 配置项（示例类别）

- 应用名、API 前缀；
- 数据库连接串（演进用）；
- MQTT：Broker、主题、凭证、ClientId 策略、是否启用；
- 是否禁用传感器模拟定时任务；
- JWT 与 CORS 列表。

## 4. 验收要点

- 配置变更无需改代码即可生效（通过 `.env` 等）；
- 生产部署文档中列出必填项与安全注意事项。
