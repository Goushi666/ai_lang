# 后端需求 · 概述与范围

## 1. 目标

后端为「井场环境监测与智能巡检」系统提供：

- **REST API**：历史查询、配置、控制指令提交、**环境分析**、**智能 Agent 对话** 等；
- **WebSocket**：向 Web 端推送传感器、车辆状态、告警等实时消息（可选推送分析摘要）；
- **北向数据接入**：通过 MQTT 接收硬件/网关上报的传感器等数据（具体 Broker 与主题以配置为准）。

## 2. 范围边界

### 2.1 在本后端职责内

- 暴露与 Web 端约定一致的 HTTP/WebSocket 接口；
- 解析、校验并持久化（或 MVP 阶段内存存储）业务数据；
- 将实时数据推送给已连接的 WebSocket 客户端；
- 可配置的跨域、MQTT、JWT 等运行参数。

### 2.2 不在本后端职责内（由其他文档/组件承担）

- 浏览器 UI 与交互（见 Web 端需求/设计）；
- 车载固件、摄像头推流协议细节（见硬件端文档，后端仅按约定对接）；
- 生产环境完整高可用、集群编排（可在演进需求中描述目标）。

## 3. 与 Web、硬件的协作关系

- **Web 端**：通过 REST 拉取历史/配置，通过 WebSocket 接收推送；接口路径与消息类型需与 [`../Web端需求文档.md`](../Web端需求文档.md) 第 4 节一致或可版本化扩展。
- **硬件端**：经 MQTT 上报传感器等数据；后端订阅配置主题并规范化写入内部模型后广播（见 `05-实时通信与MQTT.md`）。

## 4. 实现阶段说明（MVP）

当前代码库允许以 **内存仓库** 实现部分能力，但**对外接口形态与字段语义**应与本 `prd/backend` 各模块需求一致，便于后续替换为数据库与真实设备网关而不破坏前端契约。
