# 后端需求 · 传感器与环境数据

## 1. 功能需求

### 1.1 最新值查询

- 提供 `GET /api/sensors/latest`，返回当前最新一条传感器采样（温度、湿度、光照、设备标识、时间戳等）。
- **无数据时**：不得返回虚构占位数据；应返回可识别的「无数据」结果（如 HTTP 404 + 明确说明），供前端展示「--」或空状态。

### 1.2 历史查询

- 提供 `GET /api/sensors/history`，支持 `start_time`、`end_time`（ISO 8601）查询时间范围内的采样列表，按时间有序。
- 时间参数缺省或非法时，应返回明确错误（4xx），避免静默错误数据。

### 1.3 导出

- 提供 `GET /api/sensors/export`，在同一时间范围内导出 **CSV**，列至少包含：时间戳、设备 ID、温度、湿度、光照。

### 1.4 数据来源

- 支持从 **MQTT** 等外部通道接收 JSON 载荷，解析并写入与 REST/WebSocket 一致的内部模型（字段别名兼容应在实现层处理）。
- **默认不应**在服务端启动时预置大量模拟曲线数据；是否启用周期性模拟推送须可通过配置关闭，避免与真实设备数据混淆。

## 2. 数据语义

- **温度**：摄氏度（℃），浮点。
- **湿度**：相对湿度（%RH），浮点。
- **光照**：照度（Lux），浮点。
- **时间戳**：统一为 UTC 或可明确文档化的时区策略，与前端展示转换约定一致。

## 3. 与 WebSocket 的协同

- 每当产生新的有效传感器采样（含 MQTT 入库），应向 WebSocket 订阅方推送类型为 `sensor_data` 的消息，载荷字段与 REST 模型语义一致（见总体设计文档中的消息格式）。

## 4. 验收要点

- 仅有真实上报时，latest/history/export 与推送内容一致；
- 无上报时 latest 不返回假数；
- 历史与导出在同一时间窗下记录条数与数值一致。
