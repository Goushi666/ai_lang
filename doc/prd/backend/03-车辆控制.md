# 后端需求 · 车辆控制

## 1. 功能需求

### 1.1 控制指令

- 提供 `POST /api/vehicle/control`（路径前缀与 Web 文档一致，当前实现为 `/api/vehicle`）。
- 请求体至少包含：
  - **action**：如前进、后退、左转、右转、停止等（与硬件/网关协议映射在实现层完成）；
  - **speed**：速度档位或数值；
  - **timestamp**：客户端时间戳（可选，用于排障）。
- 成功时返回明确成功响应；参数非法或不可执行时返回 4xx 与错误说明。

### 1.2 状态查询

- 提供 `GET /api/vehicle/status`，返回当前车辆状态，至少包括：
  - 运行模式（手动/自动等）；
  - 速度或左右轮速度（若设计文档有定义）；
  - 连接/在线状态（MVP 可为内存模拟或后续对接真实链路）。

## 2. 实时推送

- 车辆状态变化时，通过 WebSocket 推送类型为 `vehicle_status` 的消息，载荷字段与 `GET /status` 语义一致，减少前端轮询。

## 3. 安全与可靠性（需求层）

- 应对未连接、急停等等场景有明确错误语义，前端可据此禁用按钮或提示（与 Web 端「断连保护」需求呼应；具体由硬件或网关执行停车时，后端需传递或确认指令路径）。

## 4. 验收要点

- 控制接口与状态接口、WebSocket 推送在字段上保持一致；
- 非法 action 不被静默接受。
